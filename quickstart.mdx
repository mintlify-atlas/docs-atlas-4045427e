This guide walks you through the most common patterns in fastarena: basic allocation, transactions, ArenaVec, and checkpoint/rewind.

## Prerequisites

Add fastarena to your `Cargo.toml`:

```toml Cargo.toml
[dependencies]
fastarena = "0.1.3"
```

Requires Rust 1.66.0 or later.

---

<Steps>
  <Step title="Basic allocation">
    Create an arena and start allocating. All allocation methods return direct references — no indirection, O(1) per allocation.

    ```rust
    use fastarena::Arena;

    let mut arena = Arena::new();

    let x: &mut u64 = arena.alloc(42);
    let squares: &mut [u32] = arena.alloc_slice(0u32..8);
    let s: &str = arena.alloc_str("hello");

    // Zero-cost reset — pages stay warm, no OS calls
    arena.reset();
    ```

    <Tip>
      Use `Arena::with_capacity(bytes)` to pre-allocate a larger initial block and avoid early block chaining in high-throughput workloads.
    </Tip>
  </Step>

  <Step title="Transactions — auto-rollback on failure">
    Transactions are the key differentiator. Use `with_transaction` for a closure-based API where `Ok` commits and `Err` rolls back all allocations atomically.

    <Note>
      Each `alloc` call returns a `&mut T` that borrows the transaction. Don't hold references across subsequent allocations — use the value and let it drop before calling `alloc` again.
    </Note>

    ```rust
    use fastarena::Arena;

    let mut arena = Arena::new();

    // Ok commits, Err rolls back — all allocations are atomic
    let result: Result<u32, &str> = arena.with_transaction(|txn| {
        txn.alloc_str("fastarena");
        let score = txn.alloc(100u32);
        Ok(*score)
    });
    assert_eq!(result, Ok(100));

    // Failed transaction — everything allocated inside is gone
    arena.with_transaction(|txn| {
        txn.alloc(1u32);
        txn.alloc(2u32);
        Err("abort")  // both u32s rolled back automatically
    });

    // Infallible variant — commits even through panic
    let val = arena.with_transaction_infallible(|txn| {
        *txn.alloc(7u32) * 6
    });
    assert_eq!(val, 42);
    ```
  </Step>

  <Step title="Nested savepoints">
    Transactions nest to arbitrary depth. Drop an inner savepoint without committing to discard its work while keeping the outer transaction intact.

    ```rust
    use fastarena::Arena;

    let mut arena = Arena::new();
    let mut outer = arena.transaction();
    outer.alloc_str("top-level");

    {
        let mut inner = outer.savepoint();
        inner.alloc_str("speculative-opt");
        inner.alloc(999u32);
        // dropped without commit — inner work discarded, outer untouched
    }

    outer.alloc_str("confirmed");
    outer.commit();  // "top-level" + "confirmed" survive
    ```
  </Step>

  <Step title="ArenaVec — growable arena-backed vector">
    `ArenaVec` is a growable vector backed by arena memory. Use `finish()` to transfer ownership to the arena — no destructor runs, no copy.

    ```rust
    use fastarena::{Arena, ArenaVec};

    let mut arena = Arena::new();

    let items: &mut [u32] = {
        let mut v = ArenaVec::new(&mut arena);
        for i in 0..1024 {
            v.push(i);
        }
        v.finish()  // ArenaVec consumed, slice now arena-owned
    };

    assert_eq!(items.len(), 1024);
    assert_eq!(items[512], 512);
    ```

    Or use the `arenavec!` macro for concise initialization:

    ```rust
    use fastarena::{Arena, arenavec};

    let mut arena = Arena::new();

    let items: &mut [u32] = arenavec![in &mut arena; 0u32; 1024].finish();
    let values: &mut [u32] = arenavec![in &mut arena; 1, 2, 3].finish();
    assert_eq!(values, &mut [1, 2, 3]);
    ```
  </Step>

  <Step title="Checkpoint and rewind">
    Take an O(1) snapshot of arena state and roll back to it later — ideal for speculative work.

    ```rust
    use fastarena::Arena;

    let mut arena = Arena::new();
    arena.alloc(1u64);

    let cp = arena.checkpoint();   // O(1) — copies 3 integers

    arena.alloc(2u64);
    arena.alloc(3u64);
    arena.rewind(cp);              // 2 and 3 gone; blocks retained for reuse
    ```

    <Tip>
      `checkpoint()` and `rewind()` are the building blocks for the transaction system. Use `with_transaction` for structured rollback, and `checkpoint`/`rewind` directly when you need fine-grained control.
    </Tip>
  </Step>
</Steps>
